배운 것 정리
--------------------------------------------------
[ JTable ]
- 배열을 생성자의 인수로 갖을때는 사용이 불편함.
  즉 JTable 생성시부터 DB연동이 완료되어 있어야 한다
  생성한 이후에 변화되는 데이터를 반영하기가 쉽지 않았슴
  개선하기 위해 TavleModel 적극 활용하면, JTable 생성 이후에도 변화된 데이터를
  반영하기가 용이하고, 데이터를 사용하려는 로직을 UI코드에서 부담하지 않아도 된다
  즉 프레임측 코드에서는 오직 디자인 처리에 집중하고, 데이터에 대한 처리는 
  이외의 객체에서 처리하면 되는데, 5/20일에는 프레임측코드에서 여전히 데이터에 대한
  처리가 남아있었다

[ MVC ]
Model : 데이터, 데이터 처리 로직
View : 디자인 (UI)
Controller : Model과 View코드를 분리시키기 위해서 관여하는 중간자
